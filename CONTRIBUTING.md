# Guide de contribution

Merci de votre intérêt pour contribuer au projet Blog d'Actualités ! Ce document fournit des lignes directrices pour contribuer à ce projet.

## Comment contribuer

### Signaler des bugs

Si vous trouvez un bug, veuillez créer une issue en incluant :

1. Le titre et la description claire du problème
2. Les étapes pour reproduire le bug
3. L'environnement dans lequel vous avez rencontré le problème (navigateur, système d'exploitation)
4. Des captures d'écran si possible

### Suggérer des améliorations

Pour proposer des améliorations :

1. Créez une issue avec le préfixe "[Amélioration]"
2. Décrivez clairement votre suggestion et les avantages qu'elle apporterait
3. Si possible, proposez une implémentation

### Pull Requests

1. Créez une branche à partir de `main` pour vos modifications
2. Assurez-vous que votre code respecte les standards du projet
3. Incluez des tests si nécessaire
4. Mettez à jour la documentation si nécessaire
5. Créez une Pull Request avec une description claire de vos modifications

## Standards de code

- Utilisez TypeScript pour tout le code
- Suivez les conventions de nommage existantes
- Commentez votre code lorsque nécessaire
- Respectez l'architecture du projet

## Processus de développement

1. Choisissez une issue à traiter ou créez-en une nouvelle
2. Discutez de votre approche dans l'issue
3. Développez votre solution
4. Soumettez une Pull Request
5. Attendez la revue de code et les commentaires

## Tests

- Assurez-vous que tous les tests existants passent
- Ajoutez des tests pour les nouvelles fonctionnalités
- Vérifiez que votre code fonctionne sur différents navigateurs

## Communication

- Soyez respectueux et constructif dans vos commentaires
- Posez des questions si quelque chose n'est pas clair
- Aidez les autres contributeurs quand vous le pouvez

Merci de contribuer à améliorer ce projet !
